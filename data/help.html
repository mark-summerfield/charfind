CharFind

Enter a Search term to find matching Unicode characters.

Any number of search terms can be entered, and each term is one of three kinds. For example a single search term, say, arrow, will match all characters that have arrow in their metadata. Compare this with the multiple search term, say, arrow left? right? -down -up -north -south which means: match all characters that have arrow in their metadata and which have either left or right (or both), but which do not have any of down, up, north, or south.

So, to match a complete search term, every plain word, word, must be in the metadata, and at least one of any optional words, word?, but none of any negated words, -word.

In addition, a search term can be a decimal or hexadecimal Unicode code point. For example, 2022 will match both a bullet • (whose code point is U+2022) and a character ߦ (whose code point is U+07E6 but decimal number is 2022).

Some initial search terms are provided to help with learning and experimentation, see F2 below.

KeyAction
Double-ClickAdd the character from the double-clicked row in the Search Results Table to the Output Editor (if the character isn't already the last one there); see also Alt+A
F1Show this Help window
F2Pop up the Recent Searches—and replace the current search with the chosen one
F3Move the keyboard focus to the Search Results Table

Alt+CCopy the text from the Output Editor to the clipboard
Alt+LClear the Output Editor
Alt+OShow the Options dialog; see below
Alt+Q or EscQuit the application
Alt+SMove the keyboard focus to the Search box

The Options dialog can be used to set the overall scale of the user interface and to limit the number of items shown in the Searches and History menus.
